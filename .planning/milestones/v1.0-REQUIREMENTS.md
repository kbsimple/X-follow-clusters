## v1 Requirements — All Satisfied

### PARSE — Archive Parsing

- [x] **PARSE-01**: Parse `following.js` from X data archive export, extracting account IDs (usernames resolved via X API) — **Validated Phase 1**
- [x] **PARSE-02**: Handle edge cases (escaped Unicode, renamed/deleted accounts) with per-entry error handling and logging — **Validated Phase 1**
- [x] **PARSE-03**: Validate JSON structure before processing; fail fast with clear error if format unexpected — **Validated Phase 1**

### AUTH — API Authentication

- [x] **AUTH-01**: Environment variable storage for X API credentials (Bearer token, API key/secret, access token/secret) — **Validated Phase 1**
- [x] **AUTH-02**: Verify credentials with a lightweight endpoint (`GET /2/users/me`) before batch operations — **Validated Phase 1**
- [x] **AUTH-03**: Explore alternatives to paid X API (third-party services: Apify, Bright Data; document findings before committing to paid tier) — **Validated Phase 1**

### ENRICH — Profile Enrichment (X API)

- [x] **ENRICH-01**: Batch profile enrichment via `GET /2/users` (up to 100 user IDs per call) — **Validated Phase 2**
- [x] **ENRICH-02**: Track `x-rate-limit-remaining` and `x-rate-limit-reset` headers; implement exponential backoff with jitter — **Validated Phase 2**
- [x] **ENRICH-03**: Detect and flag suspended accounts (error code 63) and protected accounts (error code 179) — **Validated Phase 2**
- [x] **ENRICH-04**: Extract profile fields: bio, location, professional_category, pinned tweet text, follower/following counts, verified status — **Validated Phase 2** (Note: `professional_category` obtained via Phase 3 scraping; API does not expose this field)
- [x] **ENRICH-05**: Cache all API responses to disk immediately (never re-request within session) — **Validated Phase 2**

### SCRAPE — Profile Enrichment (Scraping)

- [x] **SCRAPE-01**: Supplemental profile page scraping for fields the API doesn't expose — **Validated Phase 3**
- [x] **SCRAPE-02**: TLS impersonation via `curl_cffi` to avoid fingerprinting blocks — **Validated Phase 3**
- [x] **SCRAPE-03**: Random delays (2–5s with jitter) between scraping requests — **Validated Phase 3**
- [x] **SCRAPE-04**: Check `robots.txt` before scraping; document which fields are scraped and legal basis — **Validated Phase 3**
- [x] **SCRAPE-05**: Graceful degradation when scraping is blocked (fall back to API data only) — **Validated Phase 3**

### CLUSTER — NLP Clustering

- [x] **CLUSTER-01**: Generate bio text embeddings via `sentence-transformers` (`all-MiniLM-L6-v2` default model) — **Validated Phase 4**
- [x] **CLUSTER-02**: Apply NLP clustering on embeddings with configurable algorithm (HDBSCAN or K-Means) — **Validated Phase 4**
- [x] **CLUSTER-03**: Enforce cluster size constraints: minimum 5 people, maximum 50 people per cluster — **Validated Phase 4**
- [x] **CLUSTER-04**: LLM-generated cluster names from member profiles (not just keyword extraction) — **Validated Phase 4**
- [x] **CLUSTER-05**: Anchor clustering with seed categories: Geographic, Occupation, Political Action, Entertainment — **Validated Phase 4**
- [x] **CLUSTER-06**: Discover additional categories beyond seed set based on profile content — **Validated Phase 4**
- [x] **CLUSTER-07**: Report cluster size histogram; warn if >50% of clusters have fewer than 5 members — **Validated Phase 4**
- [x] **CLUSTER-08**: Detect over-clustering and under-clustering via silhouette score; flag when score < 0.3 — **Validated Phase 4**

### REVIEW — Semi-Automated Review Flow

- [x] **REVIEW-01**: Display suggested clusters grouped by category type with member previews — **Validated Phase 5** (checkbox not updated in live REQUIREMENTS.md)
- [x] **REVIEW-02**: Show confidence scores for cluster membership (per-member assignment confidence) — **Validated Phase 5** (checkbox not updated in live REQUIREMENTS.md)
- [x] **REVIEW-03**: Support per-cluster actions: approve, reject, rename, merge with another cluster, split — **Validated Phase 5**
- [x] **REVIEW-04**: Batch actions: approve all clusters with >N members and confident names — **Validated Phase 5** (checkbox not updated in live REQUIREMENTS.md)
- [x] **REVIEW-05**: Allow deferring a cluster without blocking others ("not sure yet") — **Validated Phase 5**
- [x] **REVIEW-06**: Present cluster size distribution before review; warn if heavily skewed to small clusters — **Validated Phase 5**
- [x] **REVIEW-07**: After N approved rounds (configurable), offer to enable full automation mode — **Validated Phase 5**

### LIST — X API List Creation

- [x] **LIST-01**: Create native X API lists for approved clusters (5–50 people per list) — **Validated Phase 6**
- [x] **LIST-02**: Use `POST /2/lists` for list creation; handle naming conflicts (HTTP 409) gracefully — **Validated Phase 6**
- [x] **LIST-03**: Bulk add members via `POST /2/lists/{id}/members/add_all` (up to 100 per request) — **Validated Phase 6**
- [x] **LIST-04**: Validate list sizes against X's 5,000 member cap and 1,000 lists/account limit — **Validated Phase 6**
- [x] **LIST-05**: Verify list creation is possible with a test call before full run — **Validated Phase 6**

### EXPORT — Data Export

- [x] **EXPORT-01**: Export follower records with enrichment data and cluster assignments to Parquet — **Validated Phase 6**
- [x] **EXPORT-02**: Export final approved clusters to CSV with list name, member handles, and cluster metadata — **Validated Phase 6**

## Out of Scope

| Exclusion | Reason |
|-----------|--------|
| Real-time monitoring or notifications | One-time (or on-demand) run only |
| Posting or interacting with lists | Read/creation only |
| Integrating with other social platforms | X only |
| Auto-follow on list add | Creeps users out; violates trust |
| Scraping without TLS impersonation | Immediately blocked by X |
| TF-IDF-only clustering | Poor quality on short bios |
| Guaranteed demographic data | Overpromise; inference is estimation only |
