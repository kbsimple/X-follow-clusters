<task type="auto">
  <name>Task 2: Update get_auth() for OAuth 2.0 env vars</name>
  <files>src/auth/x_auth.py</files>
  <action>
    Update `get_auth()` in `src/auth/x_auth.py`:

    **Required env vars (must all be present OR tokens loaded from disk):**
    - `X_CLIENT_ID` — OAuth 2.0 client ID
    - `X_CLIENT_SECRET` — OAuth 2.0 client secret
    - `X_ACCESS_TOKEN` — OAuth 2.0 access token (Bearer)
    - `X_REFRESH_TOKEN` — OAuth 2.0 refresh token

    **Optional env vars:**
    - `X_BEARER_TOKEN` — App-only fallback bearer token (kept for read-only endpoints)

    Update the env var check logic. Raise `AuthError` if `X_CLIENT_ID` or `X_CLIENT_SECRET` is missing. The `X_ACCESS_TOKEN` may be absent initially if doing first-run OAuth flow — handle by returning XAuth with empty access_token for later population.

    **Important note:** During first-run OAuth flow, tokens are not in env vars but will be saved to `data/tokens.json` by the OAuth handler. get_auth() should also check for stored tokens as a fallback (see Plan 02 for save/load).

    For this task, implement env var loading only. Token persistence is Plan 02.
  </action>
