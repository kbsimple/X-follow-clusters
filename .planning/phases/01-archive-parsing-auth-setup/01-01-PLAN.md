<objective>
Parse the X data archive's `follower.js` file, extracting account IDs and usernames with robust edge-case handling.
</objective>

<context>
The X data archive exports follower data as a JavaScript-wrapped JSON file:
```javascript
window.YTD.follower.part0 = [...]
```

The parser must:
1. Strip the JS assignment prefix (`window.YTD.follower.part0 = `)
2. Parse the remaining as JSON
3. Extract `accountId` and `username` from each entry
4. Handle edge cases per-entry without crashing the run
5. Fail fast with clear error if overall structure is invalid

Reference: `timhutton/twitter-archive-parser` on GitHub handles this format correctly.

Interface to create:
```python
@dataclass
class FollowerRecord:
    account_id: str
    username: str
    raw_entry: dict  # preserved for debugging

def parse_follower_js(path: str | Path) -> list[FollowerRecord]:
    """Parse follower.js, return list of records. Raises ParseError on structural failure."""
```

Error handling requirements:
- Per-entry try/except: log failed entry index + error, continue with valid entries
- Structural failure (not JSON at all): raise `ParseError` with file path and line number
- Empty array is valid (no followers) -- return empty list, do not error
</context>
